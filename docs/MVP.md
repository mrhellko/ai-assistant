# План MVP

## Диалоговый цикл

1. Пользователь пишет или отправляет голосовое в Telegram.
2. Telegram отправляет update в `POST /api/telegram/webhook`.
3. `assistant` находит пользователя и активную тему.
4. LLM определяет намерение:
   - `reminder`;
   - `calendar_event`;
   - `web_task`;
   - `chat`;
   - `new_dialog`.
5. Backend выполняет действие и отправляет ответ через Telegram Bot API.
6. Сообщение пользователя и ответ ассистента сохраняются в БД.

## Контекст

Контекст хранится в таблицах `threads` и `messages`.

- обычный разговор идет в `topic_key=general`;
- поручения получают стабильный `topic_key`, например `steel_bar_purchase`;
- уточнения вроде "давай длиной 200мм" попадают в открытую задачу этой темы.

## Напоминания

LLM извлекает `due_at` в часовом поясе пользователя. Если время неполное, ассистент задает уточнение. Для "в обед" используется 13:00.

## Google

При первой функции, требующей Google, ассистент отправляет пользователю ссылку OAuth. Токены должны храниться зашифрованно.

## Интернет-поиск

Для MVP поиск выполняет backend или отдельный backend-controlled worker:

- backend хранит цель и контекст;
- поисковый модуль делает поиск и парсинг;
- backend сохраняет результат;
- уточнения обновляют ту же `delegated_task`.
