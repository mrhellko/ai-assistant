# Memory Module

## Цель

Хранить долговременные знания о пользователе отдельно от заметок, напоминаний и
истории диалогов.

Memory module должен быть primary storage для пользовательских знаний. Obsidian
может быть слоем экспорта, но не должен становиться основным хранилищем.

## Архитектура данных

### `knowledge_item`

- `id`
- `user_id`
- `type`
- `name`
- `description`
- `attributes_json`
- `embedding`
- `created_at`
- `updated_at`

`description` хранит естественно-языковое описание, пригодное для semantic
search. `attributes_json` хранит структурированные признаки для фильтрации.

### `knowledge_relation`

- `from_item_id`
- `relation_type`
- `to_item_id`

Связи нужны для обхода графа знаний: принадлежность, совместимость, причинные
связи, часть-целое, предпочтения пользователя и другие отношения.

## Поиск

Memory search должен поддерживать несколько стратегий:

- semantic vector search;
- structured attribute filtering;
- relation traversal;
- combined ranking.

Итоговая выдача должна ранжироваться с учетом семантической близости,
структурных фильтров, связей и свежести данных.

## Пример

Пользователь:

```text
Чем мне нарезать внешнюю резьбу?
```

Ожидаемый поиск:

1. Semantic search: `внешняя резьба`.
2. Structured filter: `operation=threading`.
3. Relation traversal: связанные инструменты, материалы, размеры, прошлые
   покупки или предпочтения пользователя.
4. Combined ranking.
5. Ответ на основе найденных `knowledge_item`, а не только общей LLM-памяти.

## Источники обновления

Memory module может создавать и обновлять знания из разных источников:

- текстовые диалоги;
- фотографии;
- результаты web-search поручений;
- явно сохраненные факты;
- импортированные заметки;
- будущие интеграции.

## Важные ограничения

- Memory не заменяет reminders.
- Memory не заменяет notes.
- Memory хранит и структурированные данные, и естественно-языковые описания.
- Obsidian является опциональным export layer.
- Перед автоматическим сохранением чувствительных данных нужен отдельный
  privacy policy и понятные правила подтверждения.
